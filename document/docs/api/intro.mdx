## API Response template

```json
{
	"statusCode": "200|404|500 and so on",
	"message": "Some message from API",
	"data": "Data that API sent back",
	"dateTime": " date and time in DD-MM-YYYY\\tHH:mm:ssSSS format",
	"messageConstants": "Some note"
}
```

## Some common status codes

- 200 Susscess.

- 400 Bad request.
- 404 Resource not found.

- 500 Internal server error.

## Authentication

The REST API uses JSON Web Token (JWT) for authentication. To authenticate your requests, you must include an HTTP header of `Authorization`
with the value `Bearer <token>`.

- Access token is used for authentication per request 
- Refresh token is used to generate new access token when it's stale 
	
OTP (One time password) is a number consits of 6 digits. Mainly used for reset password, verify account.

:::note

- Access token stale after 15 minutes.
- Refresh token stale after 7 days.

:::
